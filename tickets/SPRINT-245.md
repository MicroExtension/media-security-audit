# Sprint 245 - Dark Operator UI Shell

## Goal

Pause scanner feature development and make the web interface feel closer to the
target security-product experience before deeper workflow redesign.

## Scope

1. Introduce a dark application shell with red primary navigation.
2. Add a visible MEDIA brand mark and operator utility area.
3. Restyle common cards, metrics, tables, forms, badges, actions, and buttons.
4. Make the Operator page visually closer to a security dashboard while keeping
   existing workflows intact.
5. Keep all existing templates and routes functional.

## Acceptance Criteria

- The base navigation keeps all existing links and active states.
- The UI uses a dark workspace, red navigation, elevated cards, dark tables,
  and pill-style actions.
- Existing forms remain accessible and named.
- No scan behavior changes in this sprint.
- Unit tests cover the shell, brand mark, utility area, and dark theme hooks.
