# Sprint 4 - Local Web Interface

## Goal

Create the first local graphical interface for operators without adding browser
scan execution.

## Scope

1. Add a read-only FastAPI web app.
2. Add dashboard view models backed by the existing JSON store.
3. Show clients, missions, risk scores, scope counts, and finding counts.
4. Show mission detail with scope, findings, remediation plan, and report link.
5. Add a CLI command to start the local UI.
6. Add unit tests for view model behavior.

## Safety Constraints

- No scan execution from the browser in this sprint.
- No network activity in tests.
- The UI must reuse the same storage and reporting engine as the CLI.
- Default bind address must be `127.0.0.1`.

## Acceptance Criteria

- `media-audit web` starts a local read-only UI.
- Dashboard renders stored clients and missions.
- Mission detail renders stored scope, findings, and remediation plan.
- HTML report remains generated by the existing report renderer.
- Unit tests cover dashboard and mission view models.
