# Sprint 84 - Shared Layout Accessibility Landmarks

## Goal

Improve the shared web layout accessibility by making primary navigation,
status messages, errors, and workspace metadata clearer to assistive
technologies.

## Scope

- Add an explicit label to the primary top navigation.
- Mark success messages as polite status updates.
- Mark error messages as alerts.
- Label the footer workspace metadata region.
- Update tests and product notes.

## Acceptance Criteria

- The top navigation exposes a clear accessible label.
- Success messages use a polite live status region.
- Error messages use an alert region.
- Footer metadata has a stable accessible label.
- Existing page behavior and routing remain unchanged.

## Safety

- This sprint only changes shared web layout accessibility.
- No scanner execution is added.
- No network activity is added.
